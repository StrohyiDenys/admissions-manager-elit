# Завдання на створення програми:

Розробити вебпрограму, яка читає із файлу БД PostgreSQL (abit_elit.sql) інформацію про абітурієнтів, що подали заяви на факультет ЕЛІТ, про яких відомо:
 
| Дані | Назва поля у БД | Формат поля у БД |
| ------ | ------ | ------ |
| ID | id	| Integer (Identity)|
| Прізвище | last_name	| varchar(50) |
| Ім’я | first_name | varchar(50) |
| Бал за іспит | exam_score | numeric(5,1) |
| Номер школи | school_number | integer |


**Користувач – технічний секретар приймальної комісії** може додавати записи у базу даних про абітурієнтів, редагувати наявну інформацію та видаляти її через інтерфейс програми. Для доступу до функцій керування даними робітник має авторизуватись.

**Користувач – член приймальної комісії** може переглядати на головній сторінці перелік усіх поданих заяв та виконувати пошук інформації:
- прізвища абітурієнтів, які зараховані на ЕЛІТ (чий бал >= прохідного балу XX);
- прізвища абітурієнтів, чий бал за іспит вищий за XX та які навчались у школі № ZZ.

Значення XX та ZZ вводяться користувачем у відповідні поля поля головної форми. 
Результати відбору інформації користувач може зберегти у файл MS Word, який формується на основі шаблону Template MS Word Шаблон_Відбору_Абітурієнтів.dot.
## Обмеження проекту
- Тривалість проекту із розроблення ПП – з 13.03.2026р. до 06.06.2026р.
- Програма має бути написана мовою C# у IDE Visual Studio Code із використанням фреймворку ASP.NET Core для серверної частини та фреймворку Angular для клієнтської.
- Для роботи з PostgreSQL використовується Entity Framework Core.
- Генерація документів MS Word виконується з використанням бібліотеки Xceed DocX.
- Проектування має спиратися на моделі процесів компанії у нотації IDEF+BPMN та моделі програмного продукту відповідно до RUP у нотації UML.
- Реалізація програми буде вестись методом Scrum. Product Backlog буде поділений на 6 спринтів тривалістю по 2 тижні.
- По завершенні проекту програма повинна виконувати всі передбачені функції, мати документацію, перелік якої наведений у розділі

## Склад програмної документації
1. **Специфікація вимог**
2. **Технічний проект ПЗ**, який складається із:
- структурно-функціональної моделі процесу у нотації IDEF, до складу якої мають входити:
    + контекстна діаграма
    + діаграма декомпозиції 1-го рівня
- моделі процесу у нотації BPMN	
- моделей RUP у нотації UML:
    + модель варіантів використання
    + модель аналізу
    + модель проектування
    + модель реалізації ПЗ
 3. **Інструкції адміністратора** (щодо розгортання вебзастосунку) та **інструкції користувача**.
